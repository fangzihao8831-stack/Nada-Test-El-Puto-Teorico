# Nadatest

## What This Is

Nadatest is a web platform for preparing the Spanish theoretical driving exam (permiso B, DGT). Users practice with 30-question tests that simulate the real exam format, with per-topic progress tracking and a failed-questions review system. A content pipeline generates and validates question batches via Claude Code skills.

## Core Value

Students must be able to practice with factually accurate DGT exam questions that mirror the real test — wrong data is worse than no questions.

## Context

- Stack: Next.js 16+ / TypeScript / Tailwind CSS v4 / shadcn/ui / Supabase / Vercel
- Content pipeline: Questions generated by /generar-preguntas skill → validated by /validar-preguntas skill → saved to content/preguntas/ → ingested by question-bank.ts
- Temario: 12 official DGT topics in content/temario/tema_01.md–tema_12.md
- Reference exam data: 2,700 Todotest questions + 30 official DGT exam questions
- Signal catalog: 235 SVG signals in content/imagenes/senales/catalogo.json
- Known validation gap: In last session, validator passed a question with an incorrect trailer speed limit (80 instead of 90 km/h) — validation CHECK 4 needs strengthening

## Constraints

- **Content accuracy**: All numerical data in skill files must match temario_permiso_b_v3.md (source of truth)
- **Spanish UI**: All user-facing text in proper Spanish with accents
- **No over-engineering**: Skills must remain readable and maintainable by a single developer
- **Cost**: Validation pipeline uses Haiku (cheap) for fact-checking — do not regress to Opus-only

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Scenario-first question generation | DGT asks what you DO, not what you KNOW — avoids trivia questions | ✓ Good |
| Per-tema subagent validation strategy | Isolates context per topic, prevents cross-contamination | ✓ Good |
| Haiku for validation (not Opus) | 15x cheaper; Haiku sufficient for structured fact-checking | ✓ Good |
| datos-numericos.md as single source of truth for numbers | Prevents divergence between generation and validation reference data | — Pending |
| Separate datos-referencia.md for validator | Keeps validator self-contained | ⚠️ Revisit — two files can drift apart again |
